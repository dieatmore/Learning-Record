###### 稀疏数组，与java的区别

与java数组占用连续内存不同，js如何基于索引迅速定位元素？稀疏数组
js数组底层数据结构，就像以索引为键，以元素为值的键值对map对象。
比如连续存在元素的没有`洞`的基本数据类型数组，V8引擎会优化为稠密数组，使用连续内存，索引为偏移量，与java相同。
比如一个数组除一个索引位置有值，其他是undefined，是稀疏数组。则以索引hash值为键存储，但因为hash值计算速度仍然极快，与连续内存一样，也属于0(1)时间复杂度。
但对象数组即使填满也不行，仍是稀疏数组。

###### 移除大数组中多个指定索引元素？splice()函数，还是自己比较

浏览器针对splice()函数有优化，但如果移除多个元素，每次均需要元素前移重组数组，因此时间复杂度为0(n)，删除量大时影响效率。
自己比较，创建新数组，压入。
如果删除元素较少，即需要保留大量连续元素，每次push均会操作内存，影响效率。
最快是fori循环(可以提自己测过fori/forof/forEach()效率，ppt里有)，自己比较，记录开始/结束索引，一次性压入。
newArray.push(...arr.slice(start, end));

###### 模拟实现Promise.all()

```js
const myAll = function (promises) {
  return new Promise((resolve, reject) => {
    if (promises === null || typeof promises[Symbol.iterator] !== 'function') {
      return reject(new TypeError('argment is not iterable.'))
    }

    const results = []
    const pArray = Array.from(promises)
    if (pArray.length === 0) {
      return resolve(results)
    }
    let count = 0
    pArray.forEach((pro, index) => {
      Promise.resolve(pro)
        .then((r) => {
          results[index] = r
          count++
          if (count === promises.length) {
            resolve(results)
          }
        })
        .catch(reject)
    })
  })
}
```
