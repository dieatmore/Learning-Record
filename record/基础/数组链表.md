## 数组/链表

### 数组 (顺序表)

数组是一种在内存中 **连续存储** 的线性数据结构，所有元素按「**固定顺序**」排列在连续的内存地址上，通过「**下标（索引）**」可以 **直接访问** 任意位置的元素。

+ 1. **连续内存：** 数组( 顺序表 )利用 **地址连续** 的存储单元依次存储线性表中的数据元素，使得逻辑上相邻的两个元素物理位置也相邻。

<br/>

+ 2. **随机访问：** 使用 base_address + k * type_size 来通过 **偏移量** 来 **直接访问** 数组中的某元素。
例如：有存放 int 类型的数组 a, 其基准地址是 0x005，那么该数组下标为 3 的元素可通过 0x002 + 3 * 4 计算得到a[3]的地址为 0x00E.

<br/>

+ 3. **插入 / 删除效率低：** 由于数组逻辑相邻物理也相邻，所以中间插入 / 删除元素时，需要 **移动** 后续所有元素 **避免** 内存中有 **空位置**（时间复杂度 O (n)）

<br/>

+ 4. **大小特性：**
静态数组（如 C++）：声明时必须指定大小，无法动态扩容；
动态数组（如 JS/Java 的 ArrayList）：**看似可以无限添加元素** ，实则底层是「**满了就重新申请更大的连续内存，复制原元素**」

<hr/>

### 链表

链表是一种在内存中 **非连续存储** 的线性数据结构，由一系列「**节点（Node）**」组成，每个节点包含两部分：
「**数据域**」：存储元素的值；
「**指针域**」：存储下一个节点的内存地址（JS 中是引用），用于串联节点。
链表的元素不要求连续存储，节点之间通过「指针」建立联系，整个结构像一条 “链子”。

+ 1. **多种形式：** 其中最简单的一种是单链表（双向链表、循环链表），即线性表的链式存储。

<br/>

+ 2. **非连续内存：** 节点可以分散在内存的任意位置，靠指针串联；

<br/>

+ 3. **顺序访问：** 无法直接通过下标访问元素，必须从「头节点」开始，逐个遍历到目标节点（时间复杂度 O (n)）；
  
<br/>

+ 4. **动态大小：** 无需提前指定大小，新增 / 删除节点只需修改指针指向（时间复杂度 O (1)，找到节点后）；
  
<br/>

+ 5. **额外空间开销：** 每个节点需要存储指针，比数组多占用内存。
  
<hr/>

### 总结

1. **数组**：连续存储，下标随机访问快，但插入 / 删除慢，大小不够灵活；

2. **链表**：非连续存储，插入 / 删除快（改指针），但访问慢（只能顺序遍历），有额外指针内存开销；
   
3. **如何选择?**：**频繁「查询」选数组；频繁「插入 / 删除」选链表**
